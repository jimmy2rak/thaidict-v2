### 4.1.2 组件编写规范

1. **JSX（非 TypeScript）**：使用 `.jsx` 扩展名，通过 JSDoc 注释标注 Props 类型
2. **函数组件 + Hooks**：所有组件使用函数组件，禁止 class 组件（ErrorBoundary 除外）
3. **单一职责**：每个文件只负责一个功能模块
4. **样式方案**：inline style 对象 + CSS Variables，禁止 Tailwind/CSS Modules
5. **图标**：统一使用 `lucide-react`，`strokeWidth={1.5}`（IW 常量）
6. **状态管理**：局部状态用 `useState`，跨组件共享用 `AppContext`
7. **副作用**：所有 `useEffect` 必须有清理函数（如 unsubscribe、abort fetch）

### 4.4.2 编写规范

1. **运行时**：Deno（Supabase Edge Functions 基于 Deno Deploy）
2. **入口**：`Deno.serve(async (req) => { ... })`
3. **CORS**：必须处理 OPTIONS 预检请求
4. **认证**：从 `Authorization: Bearer <token>` 提取 JWT，用 `supabase.auth.getUser(token)` 验证
5. **数据库访问**：使用 `service_role` key 绕过 RLS
6. **环境变量**：通过 `Deno.env.get('VAR_NAME')` 读取
7. **错误响应**：统一 `{ error: string }` 格式，附带适当 HTTP 状态码

```ts
// Edge Function 模板
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2'

Deno.serve(async (req) => {
  if (req.method === 'OPTIONS') {
    return new Response('ok', { headers: corsHeaders })
  }
  try {
    const token = req.headers.get('Authorization')?.replace('Bearer ', '')
    const supabase = createClient(Deno.env.get('SUPABASE_URL')!, Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!)
    const { data: { user } } = await supabase.auth.getUser(token)
    if (!user) return new Response(JSON.stringify({ error: 'Unauthorized' }), { status: 401, headers: { ...corsHeaders, 'Content-Type': 'application/json' } })

    // ... 业务逻辑 ...

    return new Response(JSON.stringify({ data: result }), { status: 200, headers: { ...corsHeaders, 'Content-Type': 'application/json' } })
  } catch (e) {
    return new Response(JSON.stringify({ error: e.message }), { status: 500, headers: { ...corsHeaders, 'Content-Type': 'application/json' } })
  }
})
```

## 4.8 安全规范

1. **API 密钥**：永远不在前端暴露 `service_role` key，只使用 `anon` key
2. **用户密码**：通过 Supabase Auth 处理，应用不接触明文密码
3. **WebDAV 密码**：如需存储，必须加密后存入数据库
4. **RLS**：所有用户数据表必须启用 RLS，使用 `auth.uid()` 策略
5. **输入验证**：Edge Function 必须验证所有输入参数
6. **CORS**：Edge Function 必须正确设置 CORS 头
