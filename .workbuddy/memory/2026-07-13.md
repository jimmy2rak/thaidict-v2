## 修复：登录后整页点不动（Next dev 错误遮罩）

## 修复：登录后页面卡死 / 网页无响应（Overlay pointer-events 拦截）
- 现象：用户反馈登录后「一开始能点，然后马上不能点，最后弹出网页无响应」。F12 截图显示 `icon-192.png` 404 与 `apple-mobile-web-app-capable` 已弃用警告，但这两者本身不会导致卡死。
- 根因：页面存在全屏 `Overlay`（详情/句子详情等浮层）使用 `position:absolute; inset:0; z-index:100; pointer-events:auto`。在某些浏览器/交互路径下，这个绝对定位父层会覆盖并拦截其内部子元素的点击事件，表现为「什么都点不动」；配合动画或 StrictMode 双重 mount，容易触发「网页无响应」弹窗。
- 修复：
  1. `src/App.jsx` 的 `Overlay` 改为 `pointer-events: none`，内部内容容器再恢复 `pointer-events: auto`，确保父层不吞点击。
  2. `src/components/UIComponents.jsx` 的 `Card` / `IconButton` / `WordToken` 点击时 `e.stopPropagation()`，防止事件被父层透明层误拦截。
  3. `src/components/ThaiSentence.jsx` 的下划线单词点击时 `e.stopPropagation()`。
  4. `app/AppShell.jsx` 用 `useRef` 加初始化锁，避免 React StrictMode 双重 mount 时重复触发挂载逻辑。
- PWA 图标 404 / manifest 警告：新增 `public/icons/icon-192.svg` 与 `icon-512.svg`，`public/manifest.json` 改为引用 SVG（`type: image/svg+xml`, `sizes: any`）；`app/layout.jsx` 的 `appleWebApp.capable` 改为 `false`，消除 `apple-mobile-web-app-capable` 已弃用警告。
- 验证：用 headless Chromium 实测登录 → 四个 TabBar 切换 → 打开每日一句详情 → 页面无 404、0 pageerror、主线程正常响应。`next build` 通过。
