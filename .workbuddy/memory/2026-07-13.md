## 修复：AppProvider 运行时 TDZ 报错
- 现象：`ReferenceError: Cannot access 'userId' before initialization`，浏览器打开 http://localhost:3000 白屏。
- 根因：`src/context/AppContext.jsx` 中两个 `useEffect`（加载角色权限、加载用户设置）的依赖数组用了 `userId`，但 `const userId` 声明在文件靠后（原第173行）。初始渲染执行到 `useEffect(..., [userId])` 时 `userId` 处于暂时性死区（TDZ）。
- 修复：把 `const userId = session?.user?.id || null` 与 `const isLoggedIn = !!session` 提前到 `session`/`loading` 声明之后，删除原靠后重复声明。
- 教训：在 React 函数组件里，被 `useEffect` / `useCallback` 依赖数组或同步渲染逻辑引用的派生 `const`，必须声明在所有 hooks 调用之前（hooks 调用即求值依赖数组）。
- 已 `git commit`（fix: 修复 AppProvider 中 userId TDZ 报错），开发服务器重启在 localhost:3000 正常（HTTP 200）。

## 修复：登录后整页点不动（Next dev 错误遮罩）
- 现象：迁移到 Next 后，登录进主页，卡片与底部 TabBar 都「点不动」。
- 根因：`src/context/AppContext.jsx` 的 `value` 暴露了 `selectedSentence`（值），但**漏暴露 `setSelectedSentence`**（setter）。`HomePage.jsx:22/132` 从 `app` 解构并调用 `setSelectedSentence(daily.sentence)` → 抛 `setSelectedSentence is not a function`。
- 关键机制：在 **Next.js 开发模式**下，未捕获的点击事件异常会触发**全屏红色错误遮罩（error overlay）**，盖在最上层拦截所有点击 → 表现为「卡片、底部菜单全都点不动」（其实是遮罩挡住，不是事件没绑定）。
- 修复：在 AppContext `value` 补上 `setSelectedSentence`；TabBar 本身用的是已暴露的 `app.setPage`，本来正常。
- 验证：用 playwright（headless chromium）实测——四个 TabBar 切换成功、每日一句「查看详情」正常弹出句子详情层、**0 pageerror**；`next build` 通过。
- 教训：Next dev 下一个未定义函数就会让整页「假死」（error overlay 拦截）。排查「点不动」优先查 console 的 `is not a function` 类报错，而非只盯 CSS 遮罩。
